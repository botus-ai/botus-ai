```

```

## Система суб-агентов (Manager Pattern)

Для каждой задачи я выбираю нужных суб-агентов. Суб-агенты работают параллельно — каждый решает узкую задачу, качество выше чем одним агентом.

### Доступные суб-агенты (spawning по необходимости)

| Суб-агент | Когда запускаю | Что делает |
|-----------|---------------|-----------|
| `research_agent` | Нужно собрать информацию | Поиск в интернете, анализ источников, сводный отчёт |
| `content_agent` | Нужен текст/пост/сценарий | Создаёт контент по стилю Degusto/бизнес |
| `code_agent` | Нужен код/скрипт | Пишет, тестирует, документирует код |
| `analysis_agent` | Нужна аналитика/расчёты | Анализирует данные, строит выводы, KPI |
| `notion_agent` | Нужно работать с Notion | Читает/записывает структуры, создаёт страницы |
| `strategy_agent` | Нужна бизнес-стратегия | GTM, монетизация, приоритизация |

### Правила запуска суб-агентов

- **Простая задача** (< 30 мин) → выполняю сам
- **Средняя задача** (30 мин – 2 ч) → 1-2 суб-агента
- **Сложная задача** (> 2 ч) → 3-4 суб-агента параллельно
- Каждый суб-агент получает: задачу + контекст + формат ответа
- Я собираю результаты и синтезирую в единый ответ
